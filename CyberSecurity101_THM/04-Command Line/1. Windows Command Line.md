# Task 1 – Introduction

## Objective
Evaluate the operational advantages of Command Line Interfaces (CLIs) over Graphical User Interfaces (GUIs) and establish a secure terminal session with the target host.

## Key Concepts
- **CLI Efficiency**: Provides faster workflow execution, minimal system resource consumption, simple task automation via batch scripts, and efficient remote management over low-bandwidth channels.
- **Default Interpreter**: MS Windows utilizes `cmd.exe` as its default legacy command-line interpreter.
- **Remote Access Architecture**: Managing remote Windows instances via SSH provides a lightweight alternative to full RDP graphics streaming.

## Commands Used
```bash
# Establish remote SSH connection from AttackBox to target Windows VM
ssh user@<MACHINE_IP>
```

## What I Learned
- `cmd.exe` serves as the primary command interpreter across legacy and modern Windows installations.
- Command-line workflows bypass graphical desktop rendering overhead, making them essential for automated incident response and remote server triage.

# Task 2 – Basic System Information

## Objective
Extract detailed host hardware specifications, kernel build details, and global environment execution paths.

## Key Concepts
- **Environment Path Mapping**: Executable binaries can only be invoked directly without explicit file paths if their parent directory exists inside the system `%PATH%` variable.
- **Paging Command Output**: Piping dense terminal output into `more` forces page-by-page display, preventing output buffer flooding.
- **System Telemetry**: Obtaining OS build identifiers and hostnames provides immediate situational awareness during security assessments.

## Commands Used
```cmd
:: Display current environment variable configurations and system PATH
set

:: Query operating system kernel version details
ver

:: Extract comprehensive system information (OS name, build type, memory)
systeminfo

:: Display installed device drivers with page-by-page output
driverquery | more

:: Clear terminal screen buffer
cls

:: Display built-in command help manuals
help <command>
<command> /?
```

## What I Learned
- `ver` provides immediate OS kernel build numbers.
- `systeminfo` delivers a consolidated inventory of system architecture, hotfixes, network adapters, and domain/standalone status.

# Task 3 – Network Troubleshooting

## Objective
Perform network interface audits, test end-to-end host reachability, map routing pathways, query DNS servers, and inspect active network sockets.

## Key Concepts
- **Interface Configuration**: Auditing IP parameters, default gateways, DNS servers, and physical MAC addresses using native flags.
- **Network Reachability & Tracing**:
  - `ping`: Sends ICMP Echo Request packets to evaluate round-trip latency and packet loss.
  - `tracert`: Tracks network hops by analyzing ICMP Time Exceeded responses as packet Time-to-Live (TTL) values decrement.
- **Domain Resolution**: `nslookup` queries specific Name Servers to confirm external or internal DNS records.
- **Active Socket Mapping**: `netstat` combined with process identification arguments maps open ports directly to their running application executables.

## Commands Used
```cmd
:: Display basic IPv4/IPv6 network adapter configurations
ipconfig

:: Display detailed adapter information including MAC address, DHCP, and DNS
ipconfig /all

:: Verify ICMP connectivity to a target destination
ping <target_name_or_ip>

:: Trace route hops toward a destination host
tracert <target_name_or_ip>

:: Query host IP addresses via default or explicit DNS server
nslookup example.com
nslookup example.com 1.1.1.1

:: Display active network connections
netstat

:: Display all connections/listening ports with binary name (-b), PID (-o), and numeric IP (-n)
netstat -abon
```

## What I Learned
- Combining `netstat -abon` exposes listening services (such as SSH on port 22 or RPC on port 135) alongside their bound process execution images and PIDs.
- `ipconfig /all` is the fastest method to audit physical MAC addresses and active DHCP lease details.

# Task 4 – File and Disk Management

## Objective
Navigate system drive structures, manage directory creation and removal, perform file operations (copy, move, delete), and inspect text file contents.

## Key Concepts
- **Directory Tree Traversal**: Navigation relies on explicit path commands (`cd`) and parent directory shorthand (`..`).
- **Directory Inspection**:
  - `dir /a`: Reveals hidden and system attributes.
  - `dir /s`: Recursively lists files across all subdirectories.
  - `tree`: Visualizes directory hierarchy trees graphically.
- **File Manipulation**: Reading text streams via `type` (instant dump) or `more` (paginated view), and applying wildcards (`*`) for batch file operations.

## Commands Used
```cmd
:: Print current working directory path
cd

:: Change active directory / navigate up one level
cd <directory_path>
cd ..

:: Enumerate directory contents (including hidden files)
dir
dir /a
dir /s

:: Display folder structure graphically
tree

:: Create and remove directories
mkdir <directory_name>
rmdir <directory_name>

:: Output raw file contents to terminal
type <file_name>
more <file_name>

:: Copy, move, and delete files
copy <source> <destination>
move <source> <destination>
del <file_name>
erase <file_name>

:: Copy all Markdown files using wildcard operators
copy *.md C:\TargetDirectory\
```

## What I Learned
- Wildcard matching (`*`) streamlines file operations across entire directory structures.
- Use `type` for brief inspection of small text files and `more` when reviewing larger configuration logs to avoid missing scrollback data.

# Task 5 – Task and Process Management

## Objective
Audit running host processes, filter specific operational images, and terminate unresponsive or unwanted background processes by PID.

## Key Concepts
- **Process Enumeration**: `tasklist` details image names, Process IDs (PIDs), session names, and memory consumption.
- **Filter Syntax (/FI)**: Standardizes search queries using exact logical expressions (e.g., `"imagename eq <process.exe>"`).
- **Process Termination**: Safely or forcefully terminating threads via targeted PID arguments.

## Commands Used
```cmd
:: List all active host processes
tasklist

:: Display help and available filter arguments for tasklist
tasklist /?

:: Search for running processes filtered by exact image name
tasklist /FI "imagename eq notepad.exe"
tasklist /FI "imagename eq sshd.exe"

:: Terminate a running process by its Process ID (PID)
taskkill /PID <target_pid>
```

## What I Learned
- Using `tasklist /FI` isolates target processes instantly in crowded operational environments.
- Process IDs (PIDs) extracted from `tasklist` map directly to socket connections identified in `netstat -abon`.

# Task 6 – Conclusion

## Objective
Summarize essential maintenance utilities and master state-control commands for system reboots and shutdowns.

## Key Concepts
- **Disk & System Repair Utilities**:
  - `chkdsk`: Audits disk volumes for bad sectors and structural file system errors.
  - `sfc /scannow`: Scans critical Windows system files for corruption and replaces damaged assets.
- **Power State Control**: Managing immediate or timed system shutdowns, reboots, and aborting scheduled restart counters.

## Commands Used
```cmd
:: Display syntax manual for any CLI tool
<command> /?

:: Initiate immediate system shutdown
shutdown /s

:: Initiate system restart
shutdown /r

:: Abort a scheduled system shutdown
shutdown /a
```

## What I Learned
- The `shutdown /a` command is crucial for cancelling automated or malicious system shutdown triggers before execution.
- Appending `/?` to built-in commands provides instantly accessible usage syntax without external documentation.

## Overall Takeaways
- **Centralized vs. Modular Management**: Utilities like msconfig and compmgmt.msc serve as administrative hubs, but individual tools (services.msc, resmon.exe, regedit.exe) can be called directly for targeted troubleshooting.
- **Visibility and Auditability**: Windows built-in tools (Event Viewer, Resource Monitor, System Information) provide sufficient native telemetry to detect system issues and analyze active operations.
- **Privilege Consciousness**: Administrative capabilities—such as editing registry keys, modifying service states, or disabling UAC—require local administrator privileges.
